Data Structures Study Guide: Arrays

1. Concise Definition

An array is a collection of elements stored at contiguous memory locations, accessible using an index. It provides constant-time access to elements given their index.

2. Time and Space Complexity

  • Access: O(1)
  • Search: O(n)
  • Insertion: O(n)
  • Deletion: O(n)
  • Space: O(n)

3. When to Use

Use arrays when you need fast access to elements by index, have a fixed size collection, or require contiguous memory allocation. Arrays are ideal for situations where the number of elements is known in advance and random access is frequent.

4. Visualization

5. Python Code Example

# D: Function Definition
def array_operations():
    # 1: Initialization
    arr = [10, 20, 30, 40, 50]
    
    # B: Output
    print("Original array:", arr)
    
    # A: Search (Access is a form of search)
    print("Element at index 2:", arr[2])
    
    # 5: Update
    arr[1] = 25
    
    # 2: Iteration
    for element in arr:
            # B: Output
            print(element, end=" ")
    print()
    
    # C: Auxiliary/Helper
    def array_sum(arr):
        total = 0
        for num in arr:
                # 4: Calculation
                total += num
        return total
    
    # B: Output
    print("Sum of array elements:", array_sum(arr))

# B: Output (function call is a form of output)
array_operations()

6. Mnemonic

"ACID" - Arrays are Contiguous, Indexed, and Direct access.

7. Common Operations

  • Traversal: Iterate through all elements
  • Insertion: Add an element at the end, beginning, or a specific index
  • Deletion: Remove an element from the end, beginning, or a specific index
  • Searching: Find an element (linear search, binary search if sorted)
  • Sorting: Arrange elements in a specific order (e.g., bubble sort, quicksort)
  • Reversing: Change the order of elements to the opposite
  • Slicing: Extract a portion of the array
  • Concatenation: Combine two or more arrays

Data Structures Study Guide: Linked Lists

1. Concise Definition

A linked list is a linear data structure where elements are stored in nodes. Each node contains a data field and a reference (link) to the next node in the sequence.

2. Time and Space Complexity

  • Access: O(n)
  • Search: O(n)
  • Insertion: O(1) at the beginning, O(n) at the end
  • Deletion: O(1) at the beginning, O(n) at the end
  • Space: O(n)

3. When to Use

Use linked lists when you need constant-time insertions/deletions from the beginning of the list and don't need random access to elements. They're ideal for implementing stacks, queues, and certain types of caches.

4. Visualization

5. Python Code Example

# D: Function Definition
class Node:
    def __init__(self, data):
        # 1: Initialization
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        # 1: Initialization
        self.head = None
    
    def append(self, data):
        # 1: Initialization
        new_node = Node(data)
        # 3: Comparison
        if not self.head:
            # 5: Update
            self.head = new_node
            return
        
        # 1: Initialization
        current = self.head
        # 2: Iteration
        while current.next:
            # 5: Update
            current = current.next
        # 5: Update
        current.next = new_node
    
    def display(self):
        # 1: Initialization
        current = self.head
        # 2: Iteration
        while current:
            # B: Output
            print(current.data, end=" -> ")
            # 5: Update
            current = current.next
        # B: Output
        print("None")

# 1: Initialization
linked_list = LinkedList()
# 5: Update
linked_list.append(1)
linked_list.append(2)
linked_list.append(3)
# B: Output
linked_list.display()

6. Mnemonic

"LINK" - List of Interconnected Nodes with Knots (pointers)

7. Common Operations

  • Insertion: Add a node at the beginning, end, or at a specific position
  • Deletion: Remove a node from the beginning, end, or a specific position
  • Traversal: Visit each node in the list
  • Search: Find a node with a specific value
  • Reverse: Change the direction of links to reverse the list
  • Merge: Combine two linked lists
  • Split: Divide a linked list into two
  • Detect cycle: Determine if the list has a loop

Data Structures Study Guide: Hash Tables

1. Concise Definition

A hash table is a data structure that implements an associative array abstract data type, a structure that can map keys to values. It uses a hash function to compute an index into an array of buckets or slots, from which the desired value can be found.

2. Time and Space Complexity

  • Insert: O(1) average, O(n) worst case
  • Delete: O(1) average, O(n) worst case
  • Search: O(1) average, O(n) worst case
  • Space: O(n)

3. When to Use

Use hash tables when you need fast insertion, deletion, and lookup of key-value pairs. They're ideal for implementing dictionaries, caches, symbol tables in compilers, and database indexing.

4. Visualization

5. Python Code Example

# D: Function Definition
class HashTable:
    def __init__(self, size=10):
        # 1: Initialization
        self.size = size
        self.table = [[] for _ in range(self.size)]
    
    def _hash(self, key):
        # 4: Calculation
        return hash(key) % self.size
    
    def insert(self, key, value):
        # 4: Calculation
        index = self._hash(key)
        # 2: Iteration
        for item in self.table[index]:
            # 3: Comparison
            if item[0] == key:
                # 5: Update
                item[1] = value
                return
        # 5: Update
        self.table[index].append([key, value])
    
    def get(self, key):
        # 4: Calculation
        index = self._hash(key)
        # 2: Iteration
        for item in self.table[index]:
            # 3: Comparison
            if item[0] == key:
                # B: Output
                return item[1]
        raise KeyError(key)

# 1: Initialization
ht = HashTable()
# 5: Update
ht.insert("name", "Alice")
ht.insert("age", 30)
# B: Output
print("Name:", ht.get("name"))
print("Age:", ht.get("age"))

6. Mnemonic

"HASH" - Hold And Seek Hastily (quick storage and retrieval)

7. Common Operations

  • Insert: Add a key-value pair to the hash table
  • Delete: Remove a key-value pair from the hash table
  • Search: Find a value associated with a given key
  • Update: Modify the value associated with a given key
  • Resize: Increase or decrease the size of the hash table
  • Hash: Convert a key into an index in the hash table
